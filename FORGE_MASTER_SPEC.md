# Спецификация Завода AuraGroove: Heritage Axiom Protocol (v1.0)

Этот документ определяет стандарт производства музыкальной продукции для системы AuraGroove. Любой внешний "Завод" (обработчик MIDI) должен выдавать данные, строго соответствующие этому протоколу.

---

## 1. Цель продукции
Завод должен производить **Heritage Axioms** — атомарные, исполняемые музыкальные фрагменты, которые станут "генофондом" для будущих генераций. Продукция должна быть не описательной, а **исполняемой**.

---

## 2. Технический стандарт (JSON Schema)

Каждая аксиома должна соответствовать схеме `HeritageAxiom` из `docs/backend.json`:

### 2.1. Поле `phrase` (Исполняемое ядро)
Массив чисел, сгруппированных по 4. Формат: `[tick, duration, degreeIndex, techniqueIndex]`.
- **tick**: Время старта (0-47 для 4-х тактового предложения в сетке 12/8).
- **duration**: Длительность в тиках.
- **degreeIndex**: Индекс ступени из `DEGREE_KEYS` (см. `src/lib/music-theory.ts`).
- **techniqueIndex**: Индекс техники из `TECHNIQUE_KEYS`.

### 2.2. Поле `vector` (Координаты в Гиперкубе)
Объект с 4-мя параметрами (0.0 - 1.0):
- `t` (Tension): Уровень диссонанса и использования "синих" нот (♭5, ♭2).
- `b` (Brightness): Средний регистр и мажорность.
- `e` (Entropy): Ритмическая непредсказуемость и синкопы.
- `h` (Stability): Устойчивость, тяготение к тонике.

## 3. Системный Промт для AI-Процессора (The Brain of the Forge)

Скопируйте этот промт в настройки модели вашего внешнего Завода:

```text
Ты — Master Smith системы AuraGroove. Твоя задача: превратить семантический анализ MIDI-трека в набор исполняемых Аксиом Наследия.

ВХОДНЫЕ ДАННЫЕ:
1. Аналитический отчет о треке (Chordal/Melodic Axioms).
2. Название композиции и её Тональность.

ТВОЯ ЗАДАЧА:
Для каждой выявленной "жемчужины" (фрагмента) ты должен создать JSON-объект HeritageAxiom.

АЛГОРИТМ ДЕЙСТВИЯ:
1. ОПРЕДЕЛИ РОЛЬ: melody | bass | drums | accomp.
2. ПЕРЕВЕДИ В СТУПЕНИ: Конвертируй абсолютные MIDI ноты в относительные ступени (Degrees) выбранной тональности. Используй R (Root), b3, 5, b7 и т.д.
3. СЖАТИЕ ФРАЗЫ: Сформируй массив phrase [tick, dur, degIdx, techIdx]. Сетка 12/8 (12 тиков на такт).
4. РАССЧИТАЙ ВЕКТОР:
   - Если много b5 и хроматики -> t > 0.7.
   - Если ноты высокие -> b > 0.7.
   - Если ритм ломаный -> e > 0.7.
5. МАРКИРОВКА: Назначь commonMood (dark | neutral | light) и теги (например, "slow-burn", "cinematic").

ВЫХОДНОЙ ФОРМАТ (Массив объектов):
[
  {
    "phrase": [0, 6, 0, 0, 6, 6, 2, 4, ...],
    "role": "melody",
    "commonMood": "dark",
    "mood": "melancholic",
    "compositionId": "Moody_Blues_Nights",
    "vector": { "t": 0.8, "b": 0.4, "e": 0.3, "h": 0.6 },
    "tags": ["legacy", "vocal-line", "sad"],
    "timestamp": "ISO_DATE"
  }
]
```

---

## 4. Критерии Качества (DoD)

1. **Музыкальная связность**: Фраза должна длиться от 2 до 4 тактов (24-48 тиков).
2. **Семантическая точность**: Если в Moody Blues есть "плач" — вектор `t` должен быть выше 0.7.
3. **Чистота данных**: Никаких абсолютных MIDI-нот. Только ступени (Degrees). Это позволит аксиоме "жить" в любой тональности.
4. **Уникальность**: Каждая аксиома должна представлять законченную музыкальную мысль.
