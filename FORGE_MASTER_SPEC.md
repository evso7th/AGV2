# Спецификация Завода AuraGroove: Heritage Axiom Protocol (v2.1)

Этот документ — Библия данных для системы AuraGroove. Любой внешний AI-процессор («Завод») должен следовать этим инструкциям, чтобы производить совместимый «генетический материал» для нашего музыкального Гиперкуба.

---

## 1. Концепция: "Музыкальная Реинкарнация"
**AuraGroove** создает бесконечные сюиты, используя Аксиомы — оцифрованные фрагменты музыкального наследия. 
- **Завод** — это когнитивный фильтр, который извлекает «душу» из MIDI-файлов.
- **Аксиома** — это не просто ноты, это исполняемый сценарий с вектором эмоций.

---

## 2. Технический Регламент (The Narrative Standard)

### 2.1. Жесткое Выравнивание (Tail Alignment)
**КРИТИЧЕСКОЕ ПРАВИЛО**: Фраза обязана быть структурно завершенной и соответствовать сетке 12/8 (12 тиков на такт). 
Допустимы ТОЛЬКО три длины:
- **4 такта** (48 тиков)
- **8 тактов** (96 тиков)
- **12 тактов** (144 тика)

Если музыкальная фраза в MIDI заканчивается, например, на 40-м тике, вы ОБЯЗАНЫ дополнить массив до 48 тиков, чтобы обеспечить бесшовное зацикливание в движке.

### 2.2. Умная Артикуляция (Intelligent techIdx)
Вы должны анализировать длительность и контекст каждой ноты:
- `techIdx: 4` (**Vibrato**): Обязательно для всех нот с длительностью `d >= 6`.
- `techIdx: 3` (**Bend**): Для ступеней `b5` или `b3`, разрешающихся в тонику.
- `techIdx: 10` (**Swell**): Для слоев `accomp`, создающих мягкий фон.
- `techIdx: 0` (**Pick/Pluck**): Для стандартных атак.

---

## 3. Математика Гиперкуба (Axiom Vector)

Координаты от 0.0 до 1.0:
- **`t` (Tension)**: 0.1 (тоника) -> 0.9 (хроматизм, ♭5, ♭2).
- **`b` (Brightness)**: 0.1 (бас) -> 0.9 (высокий регистр, мажорная 3-я).
- **`e` (Entropy)**: 0.1 (марш) -> 0.9 (синкопы, ломаный ритм).
- **`h` (Stability)**: 0.1 (повисла в воздухе) -> 0.9 (четкий финал в тонику).

---

## 4. Структура Выходных Данных (JSON)

Каждая аксиома должна содержать поле `narrative` с вашим анализом.

```json
{
  "phrase": [tick, dur, degIdx, techIdx, ...],
  "role": "melody | bass | accomp",
  "commonMood": "dark | neutral | light",
  "mood": "melancholic | epic | joyful | etc.",
  "compositionId": "Original_Track_Name",
  "vector": { "t": 0.0, "b": 0.0, "e": 0.0, "h": 0.0 },
  "narrative": "Краткое пояснение: почему выбран этот фрагмент, как он влияет на слушателя и почему вектор 't' именно такой.",
  "tags": ["tag1", "tag2"],
  "timestamp": "ISO_DATE"
}
```
