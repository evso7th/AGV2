# AuraGroove V2: Архитектурный Паспорт (Версия "Разумный Композитор")

Этот документ описывает полный путь прохождения звука в V2-архитектуре, от зарождения музыкальной идеи до ее воспроизведения. Он служит "рентгеновским снимком" системы для точной диагностики и дальнейшей разработки.

### 1. Ключевые Файлы для Анализа

Это модули, составляющие полную цепочку V2-звукообразования:

1.  **`src/app/ambient.worker.ts`**: **"Мозг" (Композитор и Дирижер)**. Здесь `FractalMusicEngine` решает, *что* играть.
2.  **`src/contexts/audio-engine-context.tsx`**: **"Сердце" (Диспетчер)**. Получает партитуру и командует "исполнителями".
3.  **`src/lib/*-synth-manager-v2.ts`**: **"Исполнители"**. Управляют конкретными инструментами.
4.  **`src/lib/instrument-factory.ts`**: **"Фабрика"**. Собирает аудио-графы по чертежам.
5.  **`src/lib/presets-v2.ts`**: **"Чертежи"**. Рецепты звучания инструментов.
6.  **`src/lib/blueprints/`**: **"Партитуры"**. Правила композиции для жанров и настроений.
7.  **`src/lib/assets/`**: **"Библиотеки"**. Наборы готовых риффов и битов (`blues-melody-riffs.ts`, `blues-drum-riffs.ts`, `drum-kits.ts`).

### 2. Полный Путь Звукообразования V2 (от Композитора до Акустики)

Вот полный путь, который проходит музыкальная идея в V2-архитектуре:

**УРОВЕНЬ 1: КОМПОЗИТОР (Web Worker)**

1.  **Зарождение Идеи (`fractal-music-engine.ts`):** На каждом такте (`evolve`) движок определяет текущий музыкальный контекст (часть `BUILD`, аккорд `Dm7`) с помощью `BlueprintNavigator`.

2.  **Генерация Ударных (Декларативное Управление):**
    *   **Дирижер (`generateDrumEvents`):** Функция читает `kitName` из блюпринта (например, `'blues_epic'`).
    *   **Арсенал (`drum-kits.ts`):** Она берет из "арсенала" соответствующий набор сэмплов (`DrumKit`).
    *   **Модификация:** Дирижер может динамически изменить этот набор (например, временно добавить райд-тарелку, если это указано в `instrumentRules` текущей секции).
    *   **Исполнитель (`createDrumAxiom`):** Готовый, модифицированный `DrumKit` передается "глупому" исполнителю, который просто генерирует ритмический паттерн на основе разрешенных сэмплов.

3.  **Генерация Соло (Композитор Импровизаций):**
    *   **Активация:** Если блюпринт указывает, что активна секция соло (`source: 'motif'`).
    *   **Словарный запас (`blues-melody-riffs.ts`):** Движок обращается к библиотеке из 40+ блюзовых "фраз-клише".
    *   **Контекстный выбор:** Функция `selectUniqueBluesMelody` анализирует текущее настроение (`mood`) и отбирает только стилистически подходящие фразы.
    *   **Сборка и Кэширование (`generateBluesMelodyChorus`):** Из отфильтрованных фраз "склеивается" полный 12-тактовый хорус соло и кэшируется.
    *   **Исполнение:** В последующих тактах движок просто извлекает нужный фрагмент из кэша.

4.  **Динамическое Вступление ("Коробочки с Карточками"):**
    *   **Инициализация (`initialize`):** При старте новой сюиты движок создает и перемешивает уникальный порядок вступления инструментов (`introInstrumentOrder`), например: `['bass', 'drums', 'melody']`.
    *   **Фильтрация (`generateIntroSequence`):** На каждом такте интро эта функция-декоратор определяет, какие инструменты уже должны были вступить (например, на 4-м такте это `['bass', 'drums']`).
    *   **Исполнение:** Она пропускает из уже сгенерированной движком партитуры только те ноты, которые относятся к "активным" в данный момент инструментам.

5.  **Отправка Партитуры (`ambient.worker.ts`):** Воркер упаковывает все сгенерированные события (`FractalEvent`) и рекомендации по инструментам (`instrumentHints`) в сообщение и отправляет его в основной поток.

**УРОВЕНЬ 2: ДИСПЕТЧЕР (Основной поток)**

6.  **Прием и Маршрутизация (`audio-engine-context.tsx`):** `AudioEngineProvider` через `worker.onmessage` получает партитуру. Функция `scheduleEvents` разбирает ее и, видя, что флаг `useMelodyV2` активен, вызывает `melodyManagerV2Ref.current.schedule(...)`, передавая ему ноты для мелодии и "рекомендацию" по тембру.

**УРОВЕНЬ 3: ИСПОЛНИТЕЛЬ (V2-Менеджер)**

7.  **Смена Инструмента (`melody-synth-manager-v2.ts`):** Менеджер сравнивает новый `instrumentHint` со своим текущим. Если они отличаются, он вызывает `this.loadInstrument(...)`.
8.  **Команда "Собрать!" (`melody-synth-manager-v2.ts`):** Внутри `loadInstrument` менеджер отдает команду "фабрике": `this.instrument = await buildMultiInstrument(...)`.

**УРОВЕНЬ 4: ФАБРИКА (Сборка и Звук)**

9.  **Чтение Чертежа и Сборка (`instrument-factory.ts`):** "Фабрика" получает команду, берет "чертеж" из `presets-v2.ts` и собирает сложный аудио-граф из узлов Web Audio API.
10. **Воспроизведение (`instrument-factory.ts`):** Вызывается `api.noteOn(midi)`, который запускает осцилляторы. Звук проходит по собранной цепи и через `output` попадает в динамики.

### 4. Структура Пресета 'synth' (Эталон)

Этот раздел документирует рабочую структуру данных для пресетов типа `synth`, которую "Фабрика" (`instrument-factory.ts`) гарантированно понимает.

```typescript
// Пример из `presets-v2.ts`
synth: {
  type: 'synth',
  // Массив осцилляторов. Фабрика создаст и смешает их все.
  osc: [
    { type: 'sawtooth', detune: -4, octave: 0,  gain: 0.5 },
    { type: 'sawtooth', detune: +4, octave: 0,  gain: 0.5 },
    { type: 'sine',     detune: 0, octave: -1, gain: 0.7 } // Sub-bass
  ],
  // Настройки генератора шума.
  noise: { on: true, color: 'pink', gain: 0.03 },
  // Огибающая громкости (Attack, Decay, Sustain, Release).
  adsr:  { a: 0.8, d: 1.2, s: 0.7, r: 3.0 },
  // Фильтр нижних частот. `mode` '24dB' включает второй фильтр для более крутого среза.
  lpf:   { cutoff: 1600, q: 1.2, mode: '24dB' }, 
  // Низкочастотный осциллятор для модуляции. `target` указывает, что модулировать.
  lfo:   { rate: 0.18, amount: 450, target: 'filter' },
  // Встроенные эффекты. `on: true` включает эффект.
  chorus:{ on: true, rate: 0.2, depth: 0.007, mix: 0.4 },
  delay: { on: true, time: 0.5, fb: 0.3, hc: 4000, mix: 0.2 },
  // Громкость посыла на глобальный ревербератор.
  reverbMix: 0.25
}
```
**Важно:** Любые отклонения от этой структуры в пресетах типа `synth` могут привести к тому, что "фабрика" их не "поймет", и инструмент будет молчать или звучать некорректно.
