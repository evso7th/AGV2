
# AuraGroove V2: Архитектурный Паспорт (Версия "Luminous Radiance")

Этот документ описывает финальную, эталонную архитектуру V2-движка, обеспечивающую абсолютную стабильность и чистоту звука.

## 1. Схема прохождения сигнала: "От Клика до Акустики"

```mermaid
[ USER INTERFACE ]
      | (1) Выбор: Genre, Mood, Anchor -> Клик "Play"
      v
[ AUDIO ENGINE CONTEXT ]
      | (2) Инициализация AudioContext, загрузка IR и Сэмплеров
      | (3) Запуск Web Worker (The Cloud Composer)
      v
[ WEB WORKER: SCHEDULER ]
      | (4) Генерация Suite DNA (Markov Grid, Tension Map, Dynasty)
      | (5) Tick (каждый такт) -> Вызов FME.evolve()
      v
[ FRACTAL MUSIC ENGINE (FME) ]
      | (6) Навигация по Блюпринту (Stage, Section, Chord)
      | (7) Делегирование специализированному МОЗГУ
      v
[ AMBIENT / BLUES BRAIN ]
      | (8) Выбор Аксиомы (Heritage) из Облака или Наследия
      | (9) Трансформация Аксиомы (L-Logic: Инверсия, Сдвиг, Jitter)
      | (10) Возврат массива FractalEvent[]
      v
[ AUDIO ENGINE CONTEXT ]
      | (11) SCORE_READY -> Рассылка событий по V2 Менеджерам
      v
[ INSTRUMENT FACTORY ]
      | (12) Синтез (Osc/Filter) или Sampler.noteOn()
      | (13) Суммирование -> Master Limiter (-6dB Headroom)
      v
[ SPEAKERS / RADIO ]
```

## 2. Специализированные Мозги (The Cognitive Layer)

Мозги — это «интерпретаторы», которые превращают холодные цифры FME в живую музыку.

### 2.1. AmbientBrain (Текстурный Мозг)
*   **Сфера**: Все не-блюзовые жанры (Ambient, Trance, House).
*   **Логика**: Работает с «Атласами» (География и Свет). 
*   **Механизм**: 
    *   Анализирует **Fog** (спектральную плотность) и **Depth** (гармоническую сложность).
    *   Использует «Правило Вдоха»: находит пустоты в мелодии Наследия и заполняет их спарклами или пианино.
    *   Управляет плавными морфингами тембра через `setParam`.

### 2.2. BluesBrain (Блюзовый Мозг)
*   **Сфера**: Жанр Blues.
*   **Логика**: Язык напряжений и разрешений.
*   **Механизм**:
    *   **♭5 Strategy**: Ищет «синие ноты» в аксиомах и принудительно разрешает их в 5 или 3 ступень.
    *   **Call & Response**: Генерирует парные фразы (нисходящий вопрос — восходящий ответ).
    *   **Symbiotic Bass**: Генерирует волкинг-бас в реальном времени, адаптируясь к сложности соло.

## 3. Принцип "Steel Foundation" (Ресурсная Безопасность)

Это ядро нашей победы над браузерным троттлингом и крахами.

### 3.1. Басовая Чистота (The G1 Standard)
Мы установили жесткий нижний порог баса на уровне **MIDI 31 (G1)**. Устраняет инфразвуковой гул и фазовые искажения.

### 3.2. Global Voice Registry & Stealing
Лимит активных голосов — **150**. Система «кражи голосов» гарантирует, что контекст не упадет при любой плотности аранжировки.

---

## 4. Нарративная Архитектура (Grid-Markov)

*   **Grid-Markov Convergence**: Гармония генерируется марковскими цепями (нелинейность), но жестко удерживается в рамках жанровой сетки (Ghost Grid).
*   **Genre Sovereignty**: Наследие фильтруется по метаданным. Треки из облака используются только в «своих» жанрах.
*   **Common Mood Fallback**: Автоматический подбор треков по категориям (Light/Dark/Neutral) при отсутствии точного совпадения.

---

## 5. The Stream Bridge (Background Sovereignty)

Режим **Broadcast (Radio)** обеспечивает абсолютную стабильность воспроизведения.
*   **Direct Routing**: MediaStream -> HTMLAudioElement.srcObject.
*   **Silk Start**: Плавный fade-in на уровне элемента для маскировки шумов буферизации.
*   **OS Priority**: Статус медиа-плеера предотвращает засыпание вкладки.

---

## 6. Legacy Extinction (Total Purge)

Начиная с версии 2.9.5, движка V1 больше не существует. Вся маршрутизация упрощена: **Ticks -> V2 Managers -> Factory -> Master**.
