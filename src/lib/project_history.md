
# Журнал Проекта "AuraGroove"

---

### ЗАПИСЬ: 2024-12-04 (План №142: "UI Structural Fix")

**ПРОБЛЕМА:**
Критическая синтаксическая ошибка в `aura-groove-v2.tsx`, блокирующая компиляцию приложения.

**ПРИЧИНА:**
Опечатки в JSX-разметке: теги `SelectTrigger` закрывались тегами `SelectContent`.

**РЕШЕНИЕ:**
1.  **Tag Realignment**: Исправлены все некорректно закрытые теги в компоненте `AuraGrooveV2`.
2.  **Syntax Verification**: Проведена проверка структуры JSX для предотвращения повторных ошибок компиляции.

**ВЫВОД:**
Интерфейс восстановлен. Функции записи сессий и детерминированного синтеза теперь доступны в рабочем окружении.

### ЗАПИСЬ: 2024-12-04 (План №141: "The Session Witness")

**ПРОБЛЕМА:**
Пользователь недоволен текущим качеством генерации («хрень наиполнейшая») и требует возможности записи сессии для фиксации результата.

**ПРИЧИНА:**
Отсутствие встроенного механизма захвата аудио-потока в приложении.

**РЕШЕНИЕ:**
1.  **Audio Stream Capture**: Внедрен `MediaStreamAudioDestinationNode`, подключенный к `masterGainNode`.
2.  **MediaRecorder API**: Реализована логика записи в формате `audio/webm;codecs=opus`.
3.  **UI Controls**: Добавлена кнопка записи с анимацией пульсации. При остановке записи файл автоматически скачивается на устройство.

**ВЫВОД:**
Теперь любой эксперимент (даже неудачный) может быть зафиксирован в виде файла. Это позволяет пользователю собирать доказательства «хреновости» или, наоборот, ловить редкие удачные моменты.

### ЗАПИСЬ: 2024-12-04 (План №139: "The Deterministic Synthesis")

**ПРОБЛЕМА:**
Критика эксперта: код был лишь формальной оберткой над случайными числами. Отсутствовала истинная математическая связность.

**ПРИЧИНА:**
Использование `Math.random` в качестве основного двигателя, что создавало "статистический шум", а не "музыкальную структуру".

**РЕШЕНИЕ:**
1.  **MusiNum Core**: Полный отказ от рандома в пользу детерминированных фрактальных функций. Каждая нота теперь вычисляется на основе семени и времени.
2.  **Markov 2nd Order**: Внедрена память контекста. Каждая нота выбирается с учетом двух предыдущих для обеспечения естественной мелодической речи.
3.  **L-System Phrasing**: 12-тактовый цикл теперь является результатом порождающей грамматики, обеспечивая глубокое родство тем A и A'.
4.  **1/f Fractal Timing**: Гуманизация переведена на розовый шум, имитирующий фрактальную структуру человеческого дыхания.

**ВЫВОД:**
Система перешла от "имитации" к "вычислению" Блюза. Это высшая точка алгоритмической композиции в проекте.
