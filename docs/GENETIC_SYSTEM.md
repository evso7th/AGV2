
# Генетическая Система АураОС (Genetic Evolution Engine)

## 1. Концепция "Бесконечного Наследования"
АураОС не просто генерирует случайную музыку. Она использует систему глобальной памяти, где каждое удачное музыкальное решение ("Шедевр") сохраняется в облаке и становится "предком" для будущих сессий.

## 2. Иерархия Рождения (The Pipeline)

### Шаг 1: Семантическое Семя (Lick Selection)
При запуске новой сюиты СОР (Система Обеспечения Разнообразия) выбирает базовый мелодический "Лик" из оцифрованной библиотеки.
- **Фильтр**: Лик выбирается на основе текущего `Mood`. Например, для `dark` выбираются минорные лики.

### Шаг 2: Генетическая Трансформация (Birth Transformation)
Выбранный лик проходит через одну из четырех фрактальных трансформаций в момент "Рождения" (Epoch 0):
- **Inversion**: Зеркальное отражение нот.
- **Retrograde**: Проигрывание задом наперед.
- **Transposition**: Сдвиг в другую тональность.
- **Rhythmic Jitter**: Микро-сдвиги времени.

### Шаг 3: Скрещивание (DNA Crossover)
Если в глобальном генофонде (`masterpieces`) есть предки, система выбирает случайного "родителя".
- **Bitwise Crossover**: Случайный Seed текущей сессии скрещивается с Seed-ом предка на битовом уровне (маска `0x55555555`).
- **Результат**: Рождается `Final Seed`, который несет в себе черты обоих родителей.

### Шаг 4: Посев (Sowing)
На основе `Final Seed` генерируются все структурные параметры пьесы:
- Гармоническая последовательность (цепь Маркова).
- Карта напряжений (`Tension Map`).
- Темп, ритмический пульс и выбор инструментов.

## 3. Математический Отбор (The Arbitrator)
Система постоянно оценивает красоту текущего момента через Матрицу Резонанса.
- **Порог (0.78)**: Если средний резонанс ансамбля превышает этот порог, семя текущей сессии автоматически сохраняется в облако.
- **Эволюция**: Чем больше качественных семян в базе, тем выше вероятность рождения гармоничных "потомков". Ваша гипотеза верна: со временем система должна генерировать резонансные состояния чаще (Convergence Theory).
