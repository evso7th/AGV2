
# Спецификация Завода AuraGroove: Heritage Axiom Protocol (v2.2)

Этот документ — Библия данных для системы AuraGroove. Любой внешний AI-процессор («Завод») должен следовать этим инструкциям, чтобы производить совместимый «генетический материал» для нашего музыкального Гиперкуба.

---

## 1. Концепция: "Музыкальная Реинкарнация"
**AuraGroove** создает бесконечные сюиты, используя Аксиомы — оцифрованные фрагменты музыкального наследия. 
- **Завод** — это когнитивный фильтр, который извлекает «душу» из MIDI-файлов.
- **Аксиома** — это не просто ноты, это исполняемый сценарий с вектором эмоций.

---

## 2. Технический Регламент (The Narrative Standard)

### 2.1. Жесткое Выравнивание (Strict Tail Alignment)
**КРИТИЧЕСКОЕ ПРАВИЛО**: Фраза обязана быть структурно завершенной и соответствовать сетке 12/8 (12 тиков на такт). 
Допустимы ТОЛЬКО три длины:
- **4 такта** (48 тиков)
- **8 тактов** (96 тиков)
- **12 тактов** (144 тика)

Если музыкальная фраза в MIDI заканчивается раньше, вы ОБЯЗАНЫ дополнить массив тишиной или продлить последнюю ноту до ближайшей границы (48, 96 или 144), чтобы обеспечить бесшовное зацикливание в Марковской сетке.

### 2.2. Сжатый Формат Phrase
Данные передаются как плоский массив чисел, где каждые 4 числа — это одна нота:
`[tick, duration, degIdx, techIdx]`

**Справочник индексов (Mapping):**
- **`degIdx`** (Ступень): `0:R, 1:b2, 2:2, 3:b3, 4:3, 5:4, 6:#4, 7:5, 8:b6, 9:6, 10:b7, 11:7, 12:R+8, 13:9, 14:11`
- **`techIdx`** (Техника): `0:pick, 1:sl, 2:h/p, 3:bn, 4:vb, 5:gr, 6:ds, 7:harm, 8:pluck, 9:hit, 10:swell`

### 2.3. Умная Артикуляция
Завод обязан анализировать контекст:
- **Вибрато (4)**: Обязательно для всех нот с длительностью `d >= 6`.
- **Бенд (3)**: Для ступеней `b5` или `b3`, разрешающихся в тонику.
- **Свелл (10)**: Только для роли `accomp` (пады).

---

## 3. Математика Гиперкуба (Axiom Vector)

Каждая аксиома должна иметь вектор `vector` (0.0 - 1.0):
- **`t` (Tension)**: 0.1 (тоника) -> 0.9 (хроматизм, ♭5, ♭2).
- **`b` (Brightness)**: 0.1 (бас) -> 0.9 (высокий регистр, мажорная 3-я).
- **`e` (Entropy)**: 0.1 (марш) -> 0.9 (синкопы, ломаный ритм).
- **`h` (Stability)**: 0.1 (повисла в воздухе) -> 0.9 (четкий финал в тонику).

---

## 4. Спецификация Ролей (Ensemble Symbiosis)

Для полноценной реконструкции оркестровки Завод должен подписывать партии аккомпанемента:
- **`accomp piano`**: Направляется в канал Piano.
- **`accomp strings`**, **`accomp violin`**, **`accomp guitar`**: Направляются в канал Harmony.
- **`accomp`**, **`accomp [anything_else]`**: Направляются в основной канал Accompaniment.

Это позволяет движку AuraGroove распределять ДНК по разным инструментам, создавая богатую вертикальную фактуру.
